## 4. 성능 분석

| 구분 | 평균 응답 | 비고 |
|------|---:|---|
| **읽기(GET)** | ~4ms | SQLite 인메모리 수준 |
| **쓰기(POST/PUT/DELETE)** | ~150ms | SQLite 파일 fsync 포함 |
| **bcrypt 연산(signup/login)** | 200~320ms | 비용 함수 특성상 의도된 지연 |

- GET 계열은 평균 **2~7ms** 로 매우 빠름
- 쓰기 계열(INSERT/UPDATE/DELETE)은 SQLite 디스크 동기화로 110~290ms
- Vercel + Neon PostgreSQL 환경에서는 쓰기 지연이 감소할 것으로 예상 (NullPool, 커넥션 풀링 없음 — cold start 제외)

## 5. 결론

- 전체 19개 엔드포인트 **190회 호출 중 180회 2xx 성공** (94.7%)
- 나머지 10회는 중복 합류 시나리오로 **409 정상 반환** — 실질 오류 0건
- 오류 처리 8개 시나리오 **100% 스펙 일치**
- 로컬(SQLite) 기준 모든 엔드포인트 정상 동작 확인 완료
